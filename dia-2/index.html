¿Atascado? Abre una pista (no te doy el código)
El truco del else if es que se entra en la PRIMERA condición verdadera y se ignora el resto. Mini 5 · El tablero del videojuego Media

El operador ternario: un if / else comprimido en una sola línea, pensado para elegir un valor.

el ternario tiene la forma condición ? valorSiVerdadero : valorSiFalso. Se lee como una pregunta: "¿condición? entonces esto, si no, lo otro". Sirve para elegir entre dos valores (un texto, un número), NO para ejecutar bloques largos de código. Si la decisión es simple, queda elegante; si es compleja o tiene muchos caminos, vuelve al if / else. La herramienta correcta para cada trabajo.
Estás armando el tablero (HUD) de un videojuego. Según las vidas y el puntaje del jugador, arma dos mensajes usando SOLO el operador ternario (nada de if / else en este ejercicio: ese es el reto). Luego imprímelos.
✅ Qué debe cumplir tu solución
  • Declara las variables vidas y puntaje.
  • Crea la variable estado con un ternario: si vidas es mayor que 0 → "Sigues vivo 🎮", si no → "Game Over 💀".
  • Crea la variable mensajePuntaje con otro ternario: si puntaje es 1000 o más → "¡Nuevo récord! 🏆", si no → "Sigue sumando 💪".
  • Prohibido usar if en este ejercicio. Todo con ternario.
  • Imprime estado y mensajePuntaje.

¿Atascado? Abre una pista (no te doy el código)
El ternario guarda un VALOR en una variable. Mira la forma del ejemplo: la condición va antes del ?, el valor para el caso verdadero entre el ? y el :, y el valor para el caso falso después del :. Son dos decisiones separadas (una para vidas, otra para puntaje), así que necesitas dos ternarios, cada uno en su propia variable.
🔥 Para los rápidos: el ternario anidado (y por qué tener cuidado)
Se pueden encadenar: nota >= 6 ? "Excelente" : nota >= 4 ? "Aprobado" : "Reprobado". Intenta crear un rango de puntaje con 3 niveles (por ejemplo: 2000+ "Maestro", 1000+ "Avanzado", menos "Novato"). Ojo: más de dos niveles anidados se vuelve ilegible rápido. En la vida real, si tienes 3 o más caminos, casi siempre conviene volver al else if. El ternario brilla en lo simple.

¿Atascado? Abre una pista (no te doy el código)
Piensa en una "puerta de seguridad" por cada regla, en orden. La estructura es una cadena if / else if / else: cada else if es una validación nueva, y solo se evalúa si las anteriores pasaron. 🔥 Cuando lo tengas: 4 mejoras para los más rápidos
  • Agrega una comisión de $300 si el monto es mayor a $30000 (resta extra al saldo).
  • Valida que montoSolicitado sea realmente un número (typeof + isNaN).
  • Limita el retiro máximo a $200000 por operación con un mensaje propio.
  • Usa un operador ternario para que el comprobante diga "con comisión" o "sin comisión" en una sola línea.
